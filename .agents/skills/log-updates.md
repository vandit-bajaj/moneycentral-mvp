---
name: log-updates
description: Enforces strict documentation recording before and after file edits
---

Whenever the user instructs you to write code, modify files, or run terminal commands, you must strictly perform the following two-step protocol:
1. Before taking action, open 'DEVELOPMENT_LOG.md' and append a log entry describing the task you are beginning, your planned architectural approach, and the target files.
2. After completing the task and verifying the outcome, update the log entry to status "COMPLETE" along with a brief note of changes made.
