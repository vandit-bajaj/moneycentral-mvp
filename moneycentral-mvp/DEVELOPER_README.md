## Project Overview
MoneyCentral MVP is a **multi‑user family office dashboard** built with **Next.js (App Router)** and **Supabase**. It enables authenticated users to:
- Manage family members.
- Track stock holdings per member.
- View live market prices via Yahoo Finance.
- Analyse portfolio metrics (total invested, current value, P&L).
- Get AI‑generated portfolio insights using Gemini.

The UI follows a **premium dark‑mode design** with Tailwind CSS, Indian‑currency formatting, smooth animations, and graceful empty‑state handling.

## Architecture
- **Next.js App Router** – Server‑components for API routes (`src/app/api/**/route.ts`) and client‑components for UI.
- **Supabase client** – Instantiated once (`src/lib/supabase.ts`) and reused across the app.
- **Session Management** – `supabase.auth.getSession()` + `onAuthStateChange` to maintain auth state client‑side.
- **RLS‑aware CRUD** – Every insert includes `user_id = auth.uid()`; selects are filtered automatically by Supabase policies.
- **Live Quotes** – Batch request to `/api/quote?symbols=...` returning `{symbol: price}`.
- **AI Analyzer** – Secure server‑side API (`/api/analyze`) that forwards a sanitized payload to Gemini and returns markdown.

## Database Schema
Assumes the following tables exist in Supabase (PostgreSQL). Primary keys are `uuid` generated by Supabase.
```sql
-- family_members
CREATE TABLE family_members (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid NOT NULL,               -- Supabase auth UID (FK to auth.users)
  name text NOT NULL,
  created_at timestamp with time zone DEFAULT now()
);

-- holdings
CREATE TABLE holdings (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid NOT NULL,               -- Supabase auth UID (FK)
  member_id uuid REFERENCES family_members(id) ON DELETE CASCADE,
  ticker_symbol text NOT NULL,        -- e.g. "INFY.NS"
  quantity numeric NOT NULL,
  avg_buy_price numeric NOT NULL,
  created_at timestamp with time zone DEFAULT now()
);
```
> **Note:** Columns used in the code are exactly `ticker_symbol` and `avg_buy_price` (as requested).

## Supabase RLS Policies
These policies lock data to the authenticated user.
```sql
-- family_members policies
CREATE POLICY "Members can SELECT own rows" ON family_members
  FOR SELECT USING (auth.uid() = user_id);
CREATE POLICY "Members can INSERT own rows" ON family_members
  FOR INSERT WITH CHECK (auth.uid() = user_id);
CREATE POLICY "Members can UPDATE own rows" ON family_members
  FOR UPDATE USING (auth.uid() = user_id) WITH CHECK (auth.uid() = user_id);
CREATE POLICY "Members can DELETE own rows" ON family_members
  FOR DELETE USING (auth.uid() = user_id);

-- holdings policies
CREATE POLICY "Holdings can SELECT own rows" ON holdings
  FOR SELECT USING (auth.uid() = user_id);
CREATE POLICY "Holdings can INSERT own rows" ON holdings
  FOR INSERT WITH CHECK (auth.uid() = user_id);
CREATE POLICY "Holdings can UPDATE own rows" ON holdings
  FOR UPDATE USING (auth.uid() = user_id) WITH CHECK (auth.uid() = user_id);
CREATE POLICY "Holdings can DELETE own rows" ON holdings
  FOR DELETE USING (auth.uid() = user_id);
```
Run the above SQL in the **Supabase dashboard → SQL editor**.

## Key API Routes
| Route | Method | Purpose | Important Logic |
|-------|--------|---------|-----------------|
| `src/app/api/quote/route.ts` | GET | Batch fetch live market prices. Accepts `?symbols=SYM1,SYM2`. Uses `yahooFinance.quote` with an array. Returns `{symbol: price}` JSON. |
| `src/app/api/holdings/route.ts` *(if exists)* | POST/GET | CRUD for holdings – always includes `user_id` from Supabase session. |
| `src/app/api/family-members/route.ts` | POST/GET/DELETE | Manage family members – `user_id` bound to session. |
| `src/app/api/analyze/route.ts` | POST | Accepts array of holdings, builds strict Gemini prompt, calls `GoogleGenerativeAI` with model `gemini-1.5-flash`, returns `{analysis: string}` markdown. |

## Authentication Flow
1. **Login / Signup Page** (`src/app/login/page.tsx`)
   - Uses Supabase `signInWithPassword` / `signUp` with email & password.
   - On success, stores session in Supabase client and redirects to `/`.
2. **Protected Dashboard** (`src/app/page.tsx`)
   - At component top: `const { data: { session } } = await supabase.auth.getSession();`
   - If no session, `redirect('/login')` (server‑side) or client‑side navigation.
3. **Sign‑Out** – Button calls `supabase.auth.signOut()` and redirects to `/login`.
4. **Session‑aware Inserts** – Every `insert` includes `{user_id: session?.user.id}`.

## Frontend Structure & Components
- **`src/app/page.tsx`** – Main dashboard (client component). Handles:
  - Fetching family members (`useEffect` on mount).
  - Fetching holdings per selected member.
  - Calculating **Total Invested**, **Current Value**, **Total P&L**.
  - Rendering three summary cards.
  - Rendering a **Family Members Office** section (add member UI).
  - Rendering the **Holdings table** (member column displayed).
  - Dropdown filter to view holdings of a specific member.
  - “Analyze Portfolio” button that posts holdings to `/api/analyze` and displays markdown via `react-markdown`.
- **`src/app/login/page.tsx`** – Simple email/password form with Tailwind styling.
- **Utility `formatINR`** – Located in `src/app/page.tsx` (or extracted to `src/lib/format.ts`). Formats numbers to Indian rupee notation, e.g., `₹1,23,456`.
- **Reusable Tailwind components** – Buttons, inputs, cards use consistent `bg-zinc-900`, `text-zinc-100`, hover `bg-zinc-800`, transition classes.
- **Empty State Components** – Show friendly messages when no family members or holdings exist.

## Utility Functions
```tsx
/** Convert a number to Indian Rupee format */
export const formatINR = (value: number): string => {
  if (isNaN(value)) return "₹0";
  const formatter = new Intl.NumberFormat('en-IN', {
    style: 'currency',
    currency: 'INR',
    minimumFractionDigits: 0,
    maximumFractionDigits: 2,
  });
  return formatter.format(value);
};
```
Used across cards and table cells:
```tsx
<span>{formatINR(totalInvested)}</span>
```

## Styling & UI Guidelines
- **Dark‑mode palette** – `bg-zinc-950`, `bg-zinc-900` for cards, `text-zinc-100` for body, `text-green-400` for profit, `text-rose-400` for loss.
- **Hover/Transition** – All interactive elements include `transition-colors duration-200 ease-in-out` and `hover:bg-zinc-800`.
- **Responsive Layout** – Grid with `md:grid-cols-3` for the three summary cards, `gap-6` spacing.
- **Animations** – Tailwind `animate-pulse` for loading states, `animate-fade-in` (custom keyframes) for modal entry.
- **Accessibility** – Semantic HTML, ARIA labels on form fields, focus outlines, sufficient contrast.

## Deployment (Vercel)
1. **Create a Vercel project** linked to the GitHub repo `vandit-bajaj/MoneyCentral-MVP`.
2. **Environment Variables** (Vercel > Settings > Environment Variables):
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - `GEMINI_API_KEY`
3. **Build Settings** – Framework preset **Next.js**; no custom build command needed (default `next build`).
4. **Deploy** – Vercel will run `npm install`, `npm run build`, and serve the app.
5. **Optional – Preview Deployments** – Each PR creates a preview URL automatically.

## Environment Variables
Create a `.env.local` at the project root (git‑ignored):
```
NEXT_PUBLIC_SUPABASE_URL=https://<your‑project>.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=public-anon-key
GEMINI_API_KEY=your‑gemini‑api‑key
```
The Supabase keys are safe to expose to the client (public anon key). The Gemini key **must not** be exposed; it is only used server‑side in the `/api/analyze` route.
